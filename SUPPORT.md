## Getting Help with Projects

### For Contributors

1. **Check existing documentation** - Start with the project's README and contributing guidelines
2. **Search existing issues** - Your question may have already been answered
3. **Join our Slack** - Ask questions in the appropriate project channel
4. **Attend a meeting** - Get real-time help from the community

### For Community Members & Users

If you're using one of our projects and need assistance:

1. **Check the project documentation** - Look for user guides and FAQs
2. **Open an issue** - Use our issue templates to report problems or ask questions
3. **Contact us directly** - Email civictechwr@gmail.com for general inquiries

## Reporting Issues

### Bug Reports

Found something that's not working? [Open a bug report](../../issues/new?template=bug_report.yml) with:
- Clear description of the problem
- Steps to reproduce
- Expected vs. actual behavior
- Screenshots if applicable

### Feature Requests

Have an idea to improve a project? [Submit a feature request](../../issues/new?template=feature_request.yml) describing:
- The problem you're trying to solve
- Your proposed solution
- How it benefits the community

### Accessibility Issues

Encountered an accessibility barrier? [Report an accessibility issue](../../issues/new?template=accessibility_issue.yml) so we can make our projects work for everyone.

## Security Concerns

For security-related issues, please see our [Security Policy](SECURITY.md). Do not report security vulnerabilities through public GitHub issues.

## Response Times

CivicTechWR is a volunteer-driven organization. Please be patient with response times:

- **GitHub Issues**: We aim to respond within 2 weeks
- **Pull Requests**: We aim to review within 1 week
- **Email**: Typically within a few days
- **Slack**: Usually same-day for active channels

## Code of Conduct

All interactions are governed by our [Code of Conduct](CODE_OF_CONDUCT.md). We're committed to maintaining a welcoming, inclusive community.
